Middleware

Learn how to intercept and modify requests in the DocGenie pipeline. Middleware allows you to inject custom logic into the request/response lifecycle.

Introduction

Middleware functions are functions that have access to the request object, the response object, and the next middleware function in the application’s request-response cycle. These functions can execute any code, make changes to the request and response objects, end the request-response cycle, and call the next middleware function in the stack.

example-middleware.js 
// Define a custom middleware function
const authMiddleware = (req, res, next) => {
  const token = req.headers['x-access-token'];

  if (!token) {
    return res.status(403).send({ message: 'No token provided!' });
  }

  // Proceed to the next middleware
  next();
};

export default authMiddleware;
info

Pro-tip: Order Matters

Middleware execution order is determined by the order in which they are registered. Always place authentication and validation layers before data processing or resource-heavy tasks to optimize performance and security.

Implementation

To register a middleware globally, use the app.use() function. For route-specific middleware, pass the function as an argument before the route handler.
